# 结构质量测试集

## 使用方式

对每个题目记录以下内容：

- 输入题目
- 模型输出
- 人工评分：`可用 / 勉强可用 / 不可用`
- 问题标签
- 修改建议

## 建议问题标签

- 结论不明确
- 标题不够结论化
- 一级论点重复
- 论据空泛
- 结构顺序混乱
- 偏叙述，缺判断
- 页数过多
- 页数过少
- 术语堆砌
- 与用户题目不贴合

## 好结构判断标准

- `core_message` 清晰且可复述
- 一级论点彼此独立，不互相改写
- 标题本身带判断，而不是目录词
- 每个 section 都能自然展开成 1 页
- 结构顺序符合“结论 -> 分析 -> 方案 -> 落地”或同等级别的业务逻辑

## 当前重点观察的 5 类错误

1. 标题像目录，不像结论
2. 不同 section 说的是同一件事
3. arguments 是空话，不是支撑
4. 结构顺序跳跃，没有故事线
5. 内容层扩写后偏离结构层原意
