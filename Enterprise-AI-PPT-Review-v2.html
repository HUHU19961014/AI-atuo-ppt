🔍 2. 重点变更深度分析

✅ .github/workflows/quality-gates.yml — 项目里程碑

这是本次修改中最具工程价值的变更。GitHub Actions workflow 在每次 push/PR 时自动运行：


步骤工具范围评价

CLI 测试pytesttest_cli.py 等 4 个文件良好

改进建议： ① workflow 顶层加 name: Quality Gates； ② lint 扩展到全量 Python 文件（ruff check tools/ tests/）； ③ 覆盖率扩展到 --source=tools.sie_autoppt（全模块）； ④ 添加 ruff format 格式检查步； ⑤ 添加 mypy 类型检查步。

✅ CLI 拆分 — cli.py → cli_parser.py + cli.py

原 cli.py 的 main() 函数中有大量 argparse 定义（~100 行参数定义 + 500+ 行参数解析逻辑），现已完全分离到 cli_parser.py 的 build_main_parser()。 这是正确的单一职责原则实践。


亮点：

  • build_main_parser() 可独立测试，不依赖 CLI 入口
  • --progress 全局 flag 新增，所有长时间命令均有阶段反馈
  • --patch-json 新增，支撑 v2-patch 增量编辑命令
  • emit_progress() 统一进度输出规范，写入 stderr，不污染 stdout（管道友好）

遗留问题：新增了 validate_command_name() 等辅助函数，需要确认其在测试中有覆盖。

✅ 异常体系重构 — 从裸 ValueError 到分层异常树

之前大量裸 raise ValueError("...") 的问题已被系统化解决：


异常类型基类退出码用途
CliExecutionErrorRuntimeError1通用执行失败
CliUserInputErrorCliExecutionError2用户输入错误
ClarifierRequestErrorValueErrorN/Aclarifier-web 请求校验失败
OpenAIHTTPStatusErrorOpenAIResponsesErrorN/AHTTP 错误（带 status_code/detail/route）

正确实践： cli.py 中的 CliUserInputError 触发 parser.exit(status=2)， 与 argparse 标准行为一致（参数错误退出码为 2）。ValueError → CliUserInputError 的映射清晰。

✅ web/streamlit_app.py — Web UI 预览版

新增的 Streamlit UI 提供了 Generate 和 Patch Existing Deck 两个 Tab， 通过 subprocess 调用 main.py，具备 900 秒超时保护。 这对非技术用户的入门门槛降低有重要意义。


当前状态：标记为 "P3 lightweight UI preview"， 属于 MVP 阶段。建议后续增加实时日志输出（stdout/stderr 流式显示）和进度条。
